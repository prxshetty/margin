Frequently Asked Questions

How is this different from pasting my chapter into ChatGPT?

margin is built around your existing files — it reads your manuscript, character profiles, world bible, and style guides automatically. You don't write prompts that describe your story; you write in your story and ask for targeted changes. Every suggestion is reviewed block-by-block with visible diffs, so you stay in control. ChatGPT is a blank chat window every time. margin is a writing workspace that knows what you're working on.

Does it work with Obsidian vaults?

Yes. Since margin works directly with markdown files, you can point it at your Obsidian vault folder. Your chapters, characters, and notes are read as-is — no export step, no plugin needed.

How does it decide which character files to pull in?

margin scans your workspace for markdown files and uses a context resolver that matches character references in your active chapter against your character files. If your scene mentions Kaelen, margin finds kaelen.md and includes it automatically. You can also pin specific files to the context manually.

Can I use a reasoning model like DeepSeek?

Absolutely. margin works with any OpenAI-compatible API, so you can use DeepSeek, Claude, GPT, or local models via Ollama and LM Studio. Swap models through the endpoint config — no code changes needed.
